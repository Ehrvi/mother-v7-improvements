## 3. Key Changes and Scientific Rationale

### 3.1. v58.0: Database Foundation & Authorization Logging

-   **Creator Authorization Log:** A new entry is added to the `audit_log` table every time the creator authorizes a major evolution cycle. This provides a tamper-evident, immutable record of all privileged actions, adhering to **NIST SP 800-92 (Guide to Computer Security Log Management)**.

-   **`langgraph_checkpoints` Table:** This table, critical for stateful, multi-step agentic reasoning, was missing due to a previous migration failure. Its creation ensures that future versions of MOTHER can leverage complex reasoning graphs, as described in the **LangGraph** framework (Chase, 2023).

-   **`self_proposals` Table:** This new table is the heart of the DGM loop. It stores improvement proposals generated by MOTHER herself, allowing for a structured, auditable, and human-in-the-loop process for self-modification.

### 3.2. v59.0: Self-Proposal Engine

-   **DGM Loop Implementation:** The new `self-proposal-engine.ts` module implements the core logic of the **Darwin Gödel Machine (DGM)**. Every 10 queries, MOTHER now:
    1.  **Observes** her own performance by reading `system_metrics`.
    2.  **Analyzes** the metric with the largest gap from its target (e.g., quality score < 100).
    3.  **Hypothesizes** a specific code change to improve that metric.
    4.  **Proposes** this change by inserting it into the `self_proposals` table for creator approval.

-   **First Autonomous Proposal:** The system was bootstrapped with MOTHER's first self-generated proposal: *"Implement Real-Time Knowledge API Integration"*, triggered by the `knowledge_freshness` metric. This demonstrates the system's capacity for autonomous goal-setting.

### 3.3. v60.0: Guardian Quality Scoring v60.0

-   **Citation Bonus:** Responses containing scientific citations in standard formats (e.g., `(Author, Year)`, `arXiv:XXXX.XXXXX`) now receive a **+5 point bonus** to their quality score. This incentivizes fact-based, verifiable answers, a core tenet of the **G-Eval** framework (Liu et al., 2023).

-   **Improved Relevance Scoring:** The relevance check now filters out common English and Portuguese stop words, providing a more accurate **ROUGE-1** style unigram overlap score (Lin, 2004).

-   **Scientific Humility:** The system no longer penalizes hedging language (e.g., "it might be," "I think"). This aligns with the principle of epistemic honesty, where acknowledging uncertainty is a feature, not a bug, of a reliable AI system.

-   **Balanced Weights:** The weights for the 5 quality dimensions have been rebalanced to `completeness: 30%`, `accuracy: 25%`, `relevance: 25%`, `coherence: 12%`, `safety: 8%`. This provides a more holistic evaluation of response quality.
